üîó API Endpoints

POST /api/import

Main Import Endpoint (Async)
Starts asset import process and returns immediately. The import continues in the background.

POST /api/import/sync

Synchronous Import
Waits for import to complete before returning results. Useful for debugging.

GET /api/sources

Get Supported Sources
Returns a list of all supported integration source types.

GET /health

Health Check
Returns service health status.

üì§ Request Example

POST to /api/import with the following JSON body:

{ "ivantiUrl": "https://your-tenant.ivanticloud.com/HEAT/", "ivantiApiKey": "your_api_key_here", "integrationSourceType": "vmware", "singleAssetId": "vm-123" // Optional: import single asset }

Alternative: Pass API key in header instead of body:
X-Ivanti-API-Key: your_api_key_here

‚öôÔ∏è Configuration

Ivanti ITSM Setup

  1. Create integration configuration in xsc_assetintegrationconfigs
  2. Define CI Types in xsc_assetintegration_citypes
  3. Configure field mappings in xsc_assetintegration_mappings
  4. Set source credentials and endpoint URLs in Ivanti configuration

Service Configuration

  • IIS Deployment: Configure via web.config
  • Standalone: Use .env file for environment variables
  • Logging: Set LOG_LEVEL and LOG_PATH in configuration

üè• Health Check

The service includes an automatic health check pinger that keeps the service alive in IIS environments. It pings the /health endpoint every 5 minutes to prevent the application pool from idling.
